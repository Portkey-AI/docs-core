This guide is for **developers/consumers** who were given a Portkey Gateway setup by their platform team.

## What you need from your platform team

- Portkey Gateway URL (usually `https://api.portkey.ai`)
- Portkey API key scoped for your workspace/team
- Required metadata template (for `x-portkey-metadata`)

## Configure Claude Desktop

<Steps>
  <Step title="Enable developer mode">
    In Claude Desktop, open **Help → Troubleshooting → Enable Developer mode**.
  </Step>

  <Step title="Open third-party inference settings">
    Go to **Developer → Configure third-party inference**.
  </Step>

  <Step title="Set Gateway details">
    Select **Gateway (Anthropic-compatible)** and set:

    - Gateway base URL: `https://api.portkey.ai`
    - Gateway API key: your provided Portkey key
    - Gateway auth scheme: `bearer`
  </Step>

  <Step title="Set Gateway extra header">
    Add:

    - Header: `x-portkey-metadata`
    - Value (example):

    ```json
    {"tenant":"acme","user":"dev@acme.com","team":"engineering","env":"prod"}
    ```
  </Step>

  <Step title="Apply + restart">
    Click **Apply locally**, fully quit Claude Desktop, and reopen.
  </Step>
</Steps>

## Validate your setup

1. Send a prompt from Claude Desktop.
2. Confirm with your platform team that the request appears in Portkey Logs.
3. Confirm your metadata fields are present and correct.
4. Confirm model/policy behavior matches your assigned team policy.

## Troubleshooting

- **401/403**: your API key may be expired or scoped incorrectly.
- **Policy mismatch**: confirm metadata JSON keys exactly match your team template.
- **No response/logs**: verify base URL is exactly `https://api.portkey.ai` and auth scheme is `bearer`.
