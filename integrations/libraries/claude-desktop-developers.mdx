This guide is for developers who want Claude Desktop requests to run through Portkey.

## What you achieve

After setup, you will be able to:

- Use Claude Desktop with your org-approved provider/model routing
- Have requests attributed with metadata for policy + analytics
- See outcomes in Portkey logs (and your admin can track spend/limits)

## Path A: Your platform team already set this up

Use this path if your company gave you credentials.

### You need from your platform team

- Gateway URL (usually `https://api.portkey.ai`)
- API key for your team/workspace
- Metadata JSON template for `x-portkey-metadata`

## Path B: You're doing this yourself

Use this if you are piloting solo or do not have a platform admin.

1. Add a provider integration in Portkey (Anthropic/Bedrock/Vertex).
2. Create a config with a default model route.
3. Create an API key attached to that config.
4. Then continue with the Claude Desktop steps below.

<Note>
  If needed, use provider setup docs: [Anthropic](/integrations/llms/anthropic), [Vertex AI](/integrations/llms/vertex-ai), [AWS Bedrock](/integrations/llms/bedrock/aws-bedrock).
</Note>

## Connect Portkey to Claude Desktop

<Steps>
  <Step title="Enable Developer mode">
    In Claude Desktop: **Help → Troubleshooting → Enable Developer mode**.
  </Step>

  <Step title="Open third-party inference">
    Go to **Developer → Configure third-party inference**.
  </Step>

  <Step title="Configure Gateway">
    Select **Gateway (Anthropic-compatible)** and set:

    - Gateway base URL: `https://api.portkey.ai`
    - Gateway API key: your Portkey key
    - Gateway auth scheme: `bearer`
  </Step>

  <Step title="Add metadata header">
    In **Gateway extra headers**, add:

    - Header: `x-portkey-metadata`
    - Value:

    ```json
    {"tenant":"acme","user":"dev@acme.com","team":"engineering","env":"prod"}
    ```
  </Step>

  <Step title="Apply and restart">
    Click **Apply locally**, fully quit Claude Desktop, and reopen.
  </Step>
</Steps>

## Verify the result (fruit of setup)

1. Send a real prompt in Claude Desktop.
2. Confirm with your admin that logs show your request and metadata.
3. Confirm expected model/policy behavior (limits, guardrails, allowed models).
4. Confirm usage is attributed correctly for cost tracking.

## Troubleshooting

- **401/403**: key expired or wrong scope.
- **No logs**: base URL must be exactly `https://api.portkey.ai`; auth scheme must be `bearer`.
- **Policy not applying**: metadata JSON must be valid and key names must match policy rules exactly.
