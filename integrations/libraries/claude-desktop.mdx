[Claude Desktop](https://claude.ai/download) now supports **Gateway** connections for third-party inference. With Portkey as your gateway, you can run Claude Desktop with production-ready controls:

- **Observability** — Track cost, tokens, and latency in [Logs](https://app.portkey.ai/logs)
- **Governance** — Apply budget limits, rate limits, and scoped [API keys](https://app.portkey.ai/api-keys)
- **Reliability** — Use [Configs](/product/ai-gateway/configs) for fallbacks, load balancing, retries, and caching
- **Model flexibility** — Route Claude Desktop traffic to Anthropic, Bedrock, Vertex, and more through Portkey

<Note>
  This guide uses Claude Desktop's Gateway path (Anthropic-compatible endpoint), similar to third-party inference flows.
</Note>

## 1. Setup Portkey

<Steps>
  <Step title="Add Provider">
    Go to [Model Catalog](https://app.portkey.ai/model-catalog) → **Add Provider**.

    <Frame>
      <img
        src="/Screenshot2025-07-21at5.29.57PM.png"
        width="500"
      />
    </Frame>
  </Step>

  <Step title="Configure Credentials">
    Select your provider (Anthropic, Bedrock, Vertex, OpenAI, etc.), enter credentials, and create a slug such as `anthropic-prod`.

    <Frame>
      <img
        src="/images/product/model-catalog/create-provider-page.png"
        width="500"
      />
    </Frame>
  </Step>

  <Step title="Create Config">
    Go to [Configs](https://app.portkey.ai/configs) and create:

    ```json
    {
      "override_params": {
        "model": "@anthropic-prod/claude-sonnet-4-20250514"
      }
    }
    ```

    <Note>
      For Anthropic models, add `"cache_control": {"type": "ephemeral"}` inside `override_params` to optimize repeated context costs.
    </Note>

    Save and note the Config ID.
  </Step>

  <Step title="Create Portkey API Key">
    Go to [API Keys](https://app.portkey.ai/api-keys) → **Create new key** → attach the config → save. Copy this key.
  </Step>
</Steps>

## 2. Configure Claude Desktop Gateway

1. Open **Claude Desktop**.
2. Go to **Settings → Integrations** (or the current Gateway/third-party inference settings screen).
3. Choose **Gateway (Anthropic-compatible)**.
4. Enter:
   - **Gateway base URL:** `https://api.portkey.ai`
   - **Gateway API key:** your Portkey API key
   - **Gateway auth scheme:** `bearer`
5. Save and restart Claude Desktop.
6. Send a test prompt and verify requests in [Portkey Logs](https://app.portkey.ai/logs).

## 3. Optional: Route dynamically with headers

If Claude Desktop exposes extra headers in your Gateway config, you can route requests per environment without changing keys:

- `x-portkey-provider: @anthropic-prod`
- `x-portkey-config: pp-your-config-id`

See all supported headers in [Inference Headers](/api-reference/inference-api/headers).

## 4. Why this is a strong default setup

Using Claude Desktop through Portkey gives teams:

- One control plane for desktop AI traffic
- Centralized model/provider governance
- Clear spend and latency visibility by user, key, and workspace
- Fast failover and provider switching without reconfiguring each user repeatedly
