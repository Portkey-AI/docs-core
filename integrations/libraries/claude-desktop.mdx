Claude Desktop with Portkey is typically a **two-persona rollout**:

1. **Platform Admins** set up provider integrations, keys, policies, and guardrails.
2. **Developers/Consumers** use the managed setup inside Claude Desktop.

Use the guide that matches your role.

## Quick decision guide

<Tabs>
  <Tab title="I am a Platform Admin">

Go to **[Claude Desktop for Platform Admins](/integrations/libraries/claude-desktop-platform-admins)** if you need to:

- Add providers and create org/workspace configs
- Set budgets, rate limits, and guardrails
- Define metadata policies using `x-portkey-metadata`
- Prepare rollout instructions for all Claude Desktop users

  </Tab>

  <Tab title="I am a Developer/User">

Go to **[Claude Desktop for Developers](/integrations/libraries/claude-desktop-developers)** if you need to:

- Configure Claude Desktop with your Portkey Gateway URL and key
- Apply your team’s metadata header template
- Verify routing and policy outcomes in logs

  </Tab>
</Tabs>

<Note>
  If you are doing both (pilot teams, small orgs), start with the Platform Admin guide, then complete the Developer guide as a validation pass.
</Note>
