Continue.dev is an open-source AI coding assistant for VS Code and JetBrains. Add Portkey to get:

- **1600+ LLMs** through one interface - switch providers instantly
- **Observability** - track costs, tokens, and latency for every request
- **Reliability** - automatic fallbacks, retries, and caching
- **Governance** - budget limits, usage tracking, and team access controls

This guide shows how to configure Continue.dev with Portkey in under 5 minutes.

<Note>
  For enterprise deployments across teams, see [Enterprise Governance](#3-enterprise-governance).
</Note>

# 1. Setup

<Steps>
<Step title="Add Provider">
Go to [Model Catalog](https://app.portkey.ai/model-catalog) → **Add Provider**.


</Step>

<Step title="Configure Credentials">
Select your provider (OpenAI, Anthropic, Vertex AI, Mistral, etc.), enter your API key, and create a slug like `openai-prod`.

<Frame>
<img src="/images/product/model-catalog/create-provider-page.png" width="500"/>
</Frame>
</Step>

<Step title="Get Portkey API Key">
Go to [API Keys](https://app.portkey.ai/api-keys) and generate your Portkey API key.
</Step>
</Steps>

---

# 2. Configure Continue.dev

<Steps>
<Step title="Open config.yaml">
In VS Code, open the Continue sidebar with `Cmd/Ctrl + L`. Click the agent/model dropdown at the top of the chat input, hover over **Local Config**, and click the **gear icon** ⚙️.

This opens your local config file at:
- `~/.continue/config.yaml` (macOS / Linux)
- `%USERPROFILE%\.continue\config.yaml` (Windows)

<Frame>
<img src="/images/integrations/continue-dev-config.png" width="600"/>
</Frame>
</Step>


<Step title="Add Portkey Configuration">
Paste the configuration for your provider. Continue treats Portkey as an OpenAI-compatible endpoint.

<Tabs>
  <Tab title="OpenAI (GPT-4o-mini)">
```yaml
name: My Config
version: 0.0.1
schema: v1

models:
  - name: "Portkey - GPT-4o-mini"
    provider: openai
    model: gpt-4o-mini
    apiBase: "https://api.portkey.ai/v1"
    apiKey: "YOUR_PORTKEY_API_KEY"
    requestOptions:
      headers:
        x-portkey-provider: "@openai-prod"
    roles:
      - chat
      - edit
      - apply
```
  </Tab>
  <Tab title="Anthropic (Claude Opus 4.6)">
```yaml
name: My Config
version: 0.0.1
schema: v1

models:
  - name: "Portkey - Claude Opus 4.6"
    provider: openai
    model: claude-opus-4-6
    apiBase: "https://api.portkey.ai/v1"
    apiKey: "YOUR_PORTKEY_API_KEY"
    requestOptions:
      headers:
        x-portkey-provider: "@anthropic-prod"
    roles:
      - chat
      - edit
      - apply
```
  </Tab>
  <Tab title="Google Vertex AI">
```yaml
name: My Config
version: 0.0.1
schema: v1

models:
  - name: "Portkey - Gemini 2.5 Pro (Vertex)"
    provider: openai
    model: gemini-2.5-pro
    apiBase: "https://api.portkey.ai/v1"
    apiKey: "YOUR_PORTKEY_API_KEY"
    requestOptions:
      headers:
        x-portkey-provider: "@vertex-prod"
    roles:
      - chat
      - edit
      - apply
```

<Note>
  For Claude models on Vertex AI, use the Vertex model ID format: `anthropic.claude-opus-4@20250514` and point `x-portkey-provider` to your Vertex AI provider slug.
</Note>
  </Tab>
  <Tab title="Mistral">
```yaml
name: My Config
version: 0.0.1
schema: v1

models:
  - name: "Portkey - Mistral Large"
    provider: openai
    model: mistral-large-latest
    apiBase: "https://api.portkey.ai/v1"
    apiKey: "YOUR_PORTKEY_API_KEY"
    requestOptions:
      headers:
        x-portkey-provider: "@mistral-prod"
    roles:
      - chat
      - edit
      - apply
```
  </Tab>
</Tabs>

Replace:
- `YOUR_PORTKEY_API_KEY` → your key from Step 1
- `@openai-prod` / `@anthropic-prod` / `@vertex-prod` / `@mistral-prod` → your actual provider slug from Model Catalog
</Step>

<Step title="Select the Model in Continue">
Save `config.yaml`. Continue auto-reloads on save. In the sidebar dropdown, switch from **Default Assistant** to **Local Config**, then pick your Portkey model from the list.

<Note>
  If the model does not appear, ensure you selected **Local Config** in the agent selector — not the default assistant.
</Note>
</Step>
</Steps>

Done! Monitor usage in the [Portkey Dashboard](https://app.portkey.ai/dashboard).

## Switch Providers

Change models by updating the **model** field and `x-portkey-provider` header in `config.yaml`:

```
@openai-prod/gpt-4o-mini
@anthropic-prod/claude-opus-4-6
@vertex-prod/gemini-2.5-pro
@mistral-prod/mistral-large-latest
```

All requests route through Portkey automatically.

## Add Multiple Models

You can define multiple models in `config.yaml` and switch between them from the Continue sidebar at any time:

```yaml
models:
  - name: "Portkey - GPT-4o-mini"
    provider: openai
    model: gpt-4o-mini
    apiBase: "https://api.portkey.ai/v1"
    apiKey: "YOUR_PORTKEY_API_KEY"
    requestOptions:
      headers:
        x-portkey-provider: "@openai-prod"
    roles:
      - chat
      - edit
      - apply

  - name: "Portkey - Claude Opus 4.6"
    provider: openai
    model: claude-opus-4-6
    apiBase: "https://api.portkey.ai/v1"
    apiKey: "YOUR_PORTKEY_API_KEY"
    requestOptions:
      headers:
        x-portkey-provider: "@anthropic-prod"
    roles:
      - chat
      - edit
      - apply

  - name: "Portkey - Gemini 2.5 Pro"
    provider: openai
    model: gemini-2.5-pro
    apiBase: "https://api.portkey.ai/v1"
    apiKey: "YOUR_PORTKEY_API_KEY"
    requestOptions:
      headers:
        x-portkey-provider: "@vertex-prod"
    roles:
      - chat
      - edit
      - apply

  - name: "Portkey - Mistral Large"
    provider: openai
    model: mistral-large-latest
    apiBase: "https://api.portkey.ai/v1"
    apiKey: "YOUR_PORTKEY_API_KEY"
    requestOptions:
      headers:
        x-portkey-provider: "@mistral-prod"
    roles:
      - chat
      - edit
      - apply
```

<Note>
**Want fallbacks, load balancing, or caching?** Create a [Portkey Config](/product/ai-gateway/configs), attach it to your API key, and set `x-portkey-config` to your Config ID in `requestOptions.headers`. See [Enterprise Governance](#3-enterprise-governance) for examples.
</Note>
