Retool is a low-code platform for building internal apps and automations. Use Portkey as a Custom AI Provider in Retool to route requests through 1600+ LLMs, add observability, and enforce governance controls — all without changing your Retool queries.

## Prerequisites

- [Portkey API key](https://app.portkey.ai/api-keys)
- Provider configured in [Model Catalog](https://app.portkey.ai/model-catalog)
- Retool account with resource creation permissions

<Note>
See [Retool's Custom AI Provider docs](https://docs.retool.com/data-sources/guides/connect/custom-ai-provider/resource) for more on resource setup.
</Note>

## Setup

<Steps>
<Step title="Configure Portkey">
1. Go to [Model Catalog](https://app.portkey.ai/model-catalog) → **Add Provider**.
2. Select your provider (OpenAI, Anthropic, etc.), enter your API key, and create a slug like `openai-prod`.
3. Go to [API Keys](https://app.portkey.ai/api-keys) → **Create new key** → Save.
</Step>

<Step title="Create a Custom AI Provider resource in Retool">
1. In Retool, go to **Resources** → **Create new** → **Resource**.
2. Search for **Custom AI Provider** and select it.
3. Give the resource a name (e.g., `Portkey`).
</Step>

<Step title="Configure the connection">
Fill in the following fields:

| Field | Value |
|-------|-------|
| **Base URL** | `https://api.portkey.ai/v1` |
| **API Key** | Your Portkey API key |
| **Compatible Schema** | OpenAI |
| **Model Names** | Use Portkey Model Catalog slugs (e.g., `@openai-prod/gpt-5`, `@anthropic-prod/claude-opus-4-7`) |

Portkey model slugs follow the format `@<provider-slug>/<model-id>`, where the provider slug is the one you created in Model Catalog. This routes each request to the right provider and model automatically — no extra headers needed.

<Frame>
<img src="/images/integrations/retool.png" alt="Retool Custom AI Provider resource configuration with Portkey settings" />
</Frame>
</Step>

<Step title="Save and verify">
Click **Create resource**. To verify, create a query in your app or workflow, select the Portkey resource, choose **Generate chat response**, and run it.

<Note>
Custom AI Provider resources don't have a **Test connection** button. Verify by running a query.
</Note>
</Step>
</Steps>

## Use Portkey in queries

In any Retool app or workflow, create a new query and select your Portkey resource. Choose an action type:

| Action | Description |
|--------|-------------|
| **Generate text** | Content generation, code generation, summarization |
| **Generate chat response** | Multi-turn conversations with message history |
| **Summarize text** | Condense documents or feedback |
| **Classify text** | Categorize inputs into predefined labels |
| **Extract entity from text** | Pull structured data from unstructured text |

### Example: Chat query

Set up a query with **Generate chat response** and use dynamic Retool expressions in your prompt:

```
System message:
You are a helpful assistant for our internal support team.

Prompt:
Analyze this customer feedback: {{ feedbackTextarea.value }}
```

## Advanced headers

Add these as custom headers on the resource for more control:

| Header | Value | Purpose |
|--------|-------|---------|
| `x-portkey-config` | Your Config ID | Enable fallbacks, retries, caching |
| `x-portkey-metadata` | `{"_user": "{{ currentUser.email }}"}` | Tag requests with user info |

## What you get with Portkey

- **1600+ LLMs** — Switch between OpenAI, Anthropic, Google, Mistral, and more from the same Retool resource
- **Observability** — Every query logged with latency, cost, and token usage in the [Portkey Dashboard](https://app.portkey.ai/dashboard)
- **Reliability** — Automatic retries and fallbacks keep your Retool apps running
- **Governance** — Budget limits, rate limits, and RBAC per team or workspace
- **Guardrails** — PII detection and content filtering on requests and responses
