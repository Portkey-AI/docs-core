[Cursor](https://cursor.sh) is an AI-first code editor. Add Portkey to get:

- **1600\+ LLMs** — Not just OpenAI & Anthropic
- **Observability** — Track costs, tokens, latency for every request
- **Governance** — Budget limits, rate limits, RBAC
- **Guardrails** — PII detection, content filtering

<Note>
  When using Portkey, Cursor-specific features (autocomplete, Apply from Chat, inline refactoring) require Cursor Pro/Enterprise plans.
</Note>

<Note>
  For enterprise governance, see [Enterprise Governance](#enterprise-governance).
</Note>

## 1. Setup

<Steps>
  <Step title="Add Provider">
    Go to [Model Catalog](https://app.portkey.ai/model-catalog) → **Add Provider**.

    <Frame>
      <img
        src="/Screenshot2025-07-21at5.29.57PM.png"
        width="500"
      />
    </Frame>
  </Step>
  <Step title="Configure Credentials">
    Select your provider (OpenAI, Anthropic, etc.), enter your API key, and create a slug like `anthropic-prod`.

    <Frame>
      <img
        src="/images/anthropic-integration.png"
        width="500"
      />
    </Frame>
  </Step>
  <Step title="Create Config">
    Go to [Configs](https://app.portkey.ai/configs) and create:

    ```json
    {
      "override_params": {
        "model": "@anthropic-prod/claude-opus-4-6",
        "cache_control": {
          "type": "ephemeral"
        }
      }
    }
    ```

    <Note>
      Add `cache_control` to your config to enable  caching and optimise costs for Anthropic models.
    </Note>

    Save and note the Config ID.
  </Step>
  <Step title="Get Portkey API Key">
    Go to [API Keys](https://app.portkey.ai/api-keys) → Create new key → Attach your config → Save.
  </Step>
</Steps>

## 2. Configure Cursor

1. Open **Cursor → Settings → Cursor Settings → Models**
2. Scroll to **API Keys** section
3. Enable **OpenAI API Key** toggle and enter your **Portkey API Key**
4. Enable **Override OpenAI Base URL** and enter: `https://api.portkey.ai/v1`

<Frame>
  ![](/images/libraries/cursor.png)
</Frame>

5. Under **Models** on the same page, create a custom model named `portkey_model` and ensure it is **enabled** (toggle on).

<Frame>
  ![](/images/libraries/cursor-portkey-model-add.png)
</Frame>

<Frame>
  ![](/images/libraries/cursor-portkey-model-enabled.png)
</Frame>

6. In **Chat**, open the model picker and select that custom model so every message goes through Portkey.

<Frame>
  ![](/images/libraries/portkey-model-picker.png)
</Frame>

<Note>
  **Logs and analytics:** Creating the custom model under **Models** is not enough on its own—you must also select it in **Chat**. Otherwise Cursor may send traffic on a built-in model id and those calls may not appear in your Portkey dashboard.

  **Custom model name:** Do not use a name that includes or resembles a real provider model identifier in any form Cursor might recognize (different casing, `gpt_4o` vs `gpt-4o`, substrings such as `gpt-4` inside a longer string, and so on). For example, avoid `gpt-4o`, `sample-gpt-4o`, or similar. Prefer a neutral label like `portkey_model` or `acme-ai-gateway`.
</Note>

Done! Monitor usage in the [Portkey Dashboard](https://app.portkey.ai/dashboard).
