</Note>

Portkey has built-in support for common endpoints like `/chat/completions`, `/embeddings`, and `/images/generations`. For every other provider endpoint—Cohere's `/rerank`, Deepgram's `/listen`, image or video APIs, custom REST paths—use this feature to route the request through Portkey and get full logging and analytics.

Append the provider's path to `https://api.portkey.ai/v1/`:

```sh
curl https://api.portkey.ai/v1/<provider-endpoint> \
  -H "Content-Type: application/json" \
  -H "x-portkey-api-key: $PORTKEY_API_KEY" \
  -H "x-portkey-provider: <provider-slug>" \
  -H "Authorization: Bearer $PROVIDER_API_KEY" \
  -d '{ ... }'
```

## 1. Authenticate

Pick one of three options:

<AccordionGroup>
<Accordion title="Option 1. Add the provider to Model Catalog (recommended)">

Store credentials in [Model Catalog](https://app.portkey.ai/model-catalog) and reference them with `@<slug>`. No `Authorization` header needed.

<CodeGroup>
```sh cURL
curl https://api.portkey.ai/v1/rerank \
  -H "x-portkey-api-key: $PORTKEY_API_KEY" \
  -H "x-portkey-provider: @cohere-prod"
```

```python Python
from portkey_ai import Portkey

portkey = Portkey(
    api_key="PORTKEY_API_KEY",
    provider="@cohere-prod"
)
```

```javascript JavaScript
import Portkey from "portkey-ai"

const portkey = new Portkey({
    apiKey: "PORTKEY_API_KEY",
    provider: "@cohere-prod"
})
```
</CodeGroup>

Model Catalog centralizes credentials, budget limits, rate limits, and key rotation.

</Accordion>

## 2. Make requests

Append the provider endpoint to `https://api.portkey.ai/v1/`. The request body is whatever the provider expects—Portkey forwards it unchanged.

### POST

<CodeGroup>
```sh cURL
curl https://api.portkey.ai/v1/rerank \
  -H "Content-Type: application/json" \
  -H "x-portkey-api-key: $PORTKEY_API_KEY" \
  -H "x-portkey-provider: @cohere-prod" \
  -d '{
    "model": "rerank-english-v2.0",
    "query": "What is machine learning?",
    "documents": [
      "Machine learning is a branch of AI focused on systems that learn from data.",
      "Data science involves analyzing and interpreting complex data sets."
    ]
  }'
```

```python Python
from portkey_ai import Portkey

portkey = Portkey(
    api_key="PORTKEY_API_KEY",
    provider="@cohere-prod"
)

response = portkey.post(
    "/rerank",
    model="rerank-english-v2.0",
    query="What is machine learning?",
    documents=[
        "Machine learning is a branch of AI focused on systems that learn from data.",
        "Data science involves analyzing and interpreting complex data sets."
    ]
)
```

```javascript JavaScript
import Portkey from "portkey-ai"

const portkey = new Portkey({
    apiKey: "PORTKEY_API_KEY",
    provider: "@cohere-prod"
})

const response = await portkey.post("/rerank", {
    model: "rerank-english-v2.0",
    query: "What is machine learning?",
    documents: [
        "Machine learning is a branch of AI focused on systems that learn from data.",
        "Data science involves analyzing and interpreting complex data sets."
    ]
})
```
</CodeGroup>

### GET, PUT, DELETE

<CodeGroup>
```sh cURL
# GET
curl https://api.portkey.ai/v1/collections \
  -H "x-portkey-api-key: $PORTKEY_API_KEY" \
  -H "x-portkey-provider: @cohere-prod"

# PUT
curl -X PUT https://api.portkey.ai/v1/collections/my-collection \
  -H "Content-Type: application/json" \
  -H "x-portkey-api-key: $PORTKEY_API_KEY" \
  -H "x-portkey-provider: @cohere-prod" \
  -d '{"metadata": {"description": "Updated description"}}'

# DELETE
curl -X DELETE https://api.portkey.ai/v1/collections/my-collection \
  -H "x-portkey-api-key: $PORTKEY_API_KEY" \
  -H "x-portkey-provider: @cohere-prod"
```

```python Python
portkey.get("/collections")

portkey.put(
    "/collections/my-collection",
    metadata={"description": "Updated description"}
)

portkey.delete("/collections/my-collection")
```

```javascript JavaScript
await portkey.get("/collections")

await portkey.put("/collections/my-collection", {
    metadata: { description: "Updated description" }
})

await portkey.delete("/collections/my-collection")
```
</CodeGroup>

## Configs work as usual

Pass an `x-portkey-config` header (or a saved config ID) to apply **fallback**, **loadbalance**, or **conditional** routing—exactly like on `/chat/completions`.

```sh
curl https://api.portkey.ai/v1/rerank \
  -H "Content-Type: application/json" \
  -H "x-portkey-api-key: $PORTKEY_API_KEY" \
  -H 'x-portkey-config: {
    "strategy": { "mode": "fallback" },
    "targets": [
      { "provider": "@cohere-prod" },
      { "provider": "@cohere-backup" }
    ]
  }' \
  -d '{ "model": "rerank-english-v2.0", "query": "...", "documents": ["..."] }'
```

Retries, caching, and guardrails apply the same way.

## Force the proxy on a built-in route

Portkey handles `/chat/completions`, `/embeddings`, and other built-in routes with dedicated logic. To bypass that and send the request through untouched, prefix the path with `/v1/proxy/`:

```sh
curl https://api.portkey.ai/v1/proxy/chat/completions \
  -H "Content-Type: application/json" \
  -H "x-portkey-api-key: $PORTKEY_API_KEY" \
  -H "x-portkey-provider: openai" \
  -H "Authorization: Bearer $OPENAI_API_KEY" \
  -d '{"model":"gpt-4.1-mini","messages":[{"role":"user","content":"Hello"}]}'
```

Use this only when raw passthrough is required.

## Pricing and analytics

- **Token counts** in the response come straight from the provider.
- **Cost tracking** works when Portkey knows the model's pricing. Common provider + model combinations are recognized automatically.
- For **custom models** or unusual endpoints, add the pricing in your Portkey dashboard so cost numbers stay accurate.

## Notes

- Request and response bodies are forwarded **unchanged**. Match the provider's format—not OpenAI's.
- Every request appears in your Portkey dashboard with full logs and latency.
- `PATCH` is not supported.
