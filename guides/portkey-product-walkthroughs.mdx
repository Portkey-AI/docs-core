## 1. AI Gateway

> The unified API in front of 3000+ LLMs. One endpoint, one auth scheme, full feature parity across providers — plus production primitives like fallbacks, retries, load balancing, caching, and conditional routing.

**What's covered**

- Universal API — Chat Completions, Responses API, and Messages API translated across providers.
- Configs — declarative routing, fallbacks, retries, load balancing.
- Caching (simple and semantic), conditional routing, and canary testing.
- Multimodal, realtime, batches, files, and fine-tuning endpoints.
- Reliability — timeouts, circuit breakers, and Nitro mode.

## 3. MCP Gateway and Registry

> A central control plane for every MCP server in your org. Add servers once, broker authentication, expose only approved tools, and provision access per workspace and user.

**What's covered**

- Adding internal and external MCP servers (streamable HTTP and SSE).
- Authentication modes — OAuth 2.1, header-based, pass-through headers, and none.
- Tool provisioning — exposing only approved capabilities to clients.
- Workspace-level and user-level access control.
- Connecting clients like Cursor, Claude, and VS Code using OAuth or `x-portkey-api-key`.

## 4. Agent Gateway and Registry

> The same catalog + governance pattern, applied to A2A agents. Register an agent once with its upstream URL and auth, pick which skills to expose, provision it to the right workspaces, and let end users discover and consume it.

**What's covered**

- Registering an A2A agent — name, slug, upstream URL.
- Auth modes — `none` for public agents, header-based for production.
- Skill provisioning from the agent card — toggle each skill on or off at the gateway.
- Workspace provisioning — central platform team decides who gets the agent.
- Discovering agents as an end user and retrieving the agent card JSON.

## 6. AI Catalog

> Portkey's "AI Catalog" is the umbrella term for four central registries — one per asset type your teams use. The pattern is the same across all four: the platform team adds the asset once with credentials and auth, decides who can use it, and developers consume it through a single Portkey endpoint without ever touching upstream secrets.

| Catalog | What lives there | Who manages it |
|---|---|---|
| Model Catalog | LLM providers, models, credentials | Org admins |
| Prompt Studio | Prompts, partials, tool library | Workspace members |
| MCP Registry | MCP servers (internal + external) | Org admins |
| Agent Registry | A2A agents | Org admins |

### Model Catalog

Central place to add LLM providers (OpenAI, Anthropic, Bedrock, Azure, Vertex AI, and 200+ more), store credentials once, and decide which models and which workspaces can use them. Replaces the older virtual-keys flow.
