title: "Sharing Prompts and Partials"
description: "Share prompts and partials across workspaces in your organization with read-only access."
---

<Info>
This feature is available on all Portkey [plans](https://portkey.ai/pricing).
</Info>

Share prompts and partials from one workspace to others in your organization so teams can reuse vetted templates without duplicating work. Recipient workspaces only get **read-only** access and can fork the prompt or partial into their own workspace to customize and own it.

## How sharing works

- **Read-only access** — Recipients can view and use shared prompts and partials, but cannot edit them. Only the owning workspace can publish new versions.
- **Organization or specific workspaces** — Share with everyone in the organization, or pick specific workspaces.
- **Fork to customize** — Recipients can fork a shared prompt or partial into their own workspace to make changes.
- **Dependencies are handled** — If a prompt uses partials that aren't shared yet, Portkey prompts you to share them alongside.

## Sharing a prompt

1. Open the prompt in the [Prompt Library](/product/prompt-engineering-studio/prompt-library).
2. Click **Share**.
3. Under **Access**, choose:
   - **Organization** — Visible to everyone in your organization.
   - **Specific workspaces** — Visible only to the workspaces you select.
4. Click **Share**.

<Frame>
  ![Manage sharing dialog](/images/product/prompts/prompt-sharing.png)
</Frame>

<Note>
If the prompt references any [partials](/product/prompt-engineering-studio/prompt-partial) that aren't already shared with the target workspaces, Portkey shows an option to share those partials in the same flow. Shared prompts won't render correctly without their partials.
</Note>

## Sharing a partial

Partials follow the same flow as prompts:

1. Open the partial from the **Prompt Partials** page.
2. Click on the **Share** icon on the prompt card
3. Choose **Organization** or **Specific workspaces**.
4. Click **Share**.

## Viewing shared prompts and partials

Recipients access shared items from the **Shared with me** tab in the Prompts and Prompt Partials pages. Each entry is labeled **Read only** and shows the original author and last update time.

<Frame>
  ![Shared with me tab](/images/product/prompts/prompt-shared-with-me-screen.png)
</Frame>

Use the prompt ID shown in this tab to reference the prompt directly via the [Prompt API](/product/prompt-engineering-studio/prompt-api).

## Forking a shared prompt

Forking creates an editable copy of a shared prompt or partial in your workspace. The fork is a new prompt with its own ID and version history — changes to the original do not propagate to forks.

To fork:

1. Open the shared prompt from the **Shared with me** tab.
2. Click **Fork**.
3. Select the destination workspace.

### Requirements for forking

The destination workspace must have access to the same LLM integration the prompt uses. Integration access is managed through [Workspace Provisioning](/product/model-catalog/integrations#1-workspace-provisioning) in the Model Catalog.

<Warning>
If the LLM integration referenced by the prompt isn't provisioned to the destination workspace, the fork will fail with an error. Provision the integration to the workspace before forking, or fork into a workspace that already has access.
</Warning>

If the prompt depends on partials, those partials must also be shared with the destination workspace before forking succeeds.

## Permissions

- Any workspace member with edit access to a prompt or partial can share it.
- Only the owning workspace can publish new versions of a shared item.
- Recipients with read-only access can view, use via the Prompt API, and fork — but cannot edit the original.
