## Overview

Portkey has three API key types. Each type controls which scopes can be assigned to it.

| Key Type | Internal Type | Level |
|---|---|---|
| **Admin API Key** | `organisation-service` | Organisation-wide; crosses all workspaces |
| **Workspace Service Key** | `workspace-service` | Single workspace; automated/service use |
| **Workspace User Key** | `workspace-user` | Single workspace; individual user |

`completions.write`, `logs.write`, `prompts.render`, `mcp.invoke`, `agents.invoke`, and `guardrails.invoke` are **only available on workspace keys** (service and user). Admin keys cannot call data-plane endpoints.

The only difference between a Workspace Service Key and a Workspace User Key is that service keys can also be assigned `workspace_service_api_keys.*` and `workspace_user_api_keys.*` scopes for key management.
